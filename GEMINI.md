# GEMINI.md - DNS Block & Redirect Configurer

## 🎯 Обзор проекта
Проект представляет собой автоматизированный инструмент на Java для настройки правил блокировки и перенаправления (redirect) DNS в сервисах **Cloudflare Zero Trust** и **NextDNS**. 

Основные возможности:
- Загрузка списков доменов из внешних `hosts`-файлов по URL.
- Автоматическая фильтрация: домены с IP `0.0.0.0` или `127.0.0.1` идут в бан-лист, остальные — в список перенаправлений.
- Управление правилами через API провайдеров (Cloudflare/NextDNS).
- Поддержка запуска через GitHub Actions для периодического обновления списков.

## 🛠 Технологический стек
- **Язык**: Java 25 (используются preview-фичи: `main` без `public static`, расширенные `switch`).
- **Фреймворк**: Spring Context (IoC/DI).
- **Сборка**: Maven.
- **Библиотеки**:
  - `Lombok`: генерация кода (getters, setters, etc.).
  - `Gson`: работа с JSON.
  - `java.net.http.HttpClient`: HTTP-запросы к API.

## 🚀 Сборка и запуск

### Требования
- JDK 25 (с включенными Preview features).
- Maven 3.9+.

### Команды
- **Сборка проекта**:
  ```bash
  mvn clean install
  ```
- **Запуск локально** (после установки переменных окружения):
  ```bash
  java --enable-preview -jar target/dns-block-and-redirect-configurer-1.0-SNAPSHOT.jar
  ```

## ⚙️ Конфигурация
Проект настраивается исключительно через **переменные окружения**:

| Переменная | Описание | Значения |
| :--- | :--- | :--- |
| `DNS` | Выбор провайдера | `CLOUDFLARE` или `NEXTDNS` |
| `AUTH_SECRET` | Секретный токен/ключ API | См. README для получения |
| `CLIENT_ID` | ID аккаунта или профиля | См. README для получения |
| `BLOCK` | Ссылки на списки блокировки | URL через запятую |
| `REDIRECT` | Ссылки на списки редиректов | URL через запятую |

## 📂 Структура проекта
- `com.novibe.App`: Точка входа. Использует динамический выбор пакета Spring в зависимости от провайдера.
- `com.novibe.common`: Общая логика.
    - `config`: Spring конфигурация (`AppConfig`) и парсинг переменных окружения.
    - `data_sources`: Загрузчики и парсеры списков (`ListLoader`).
    - `util`: Логирование и вспомогательные функции.
- `com.novibe.dns`: Реализации для конкретных провайдеров.
    - `cloudflare`: Интеграция с Cloudflare Gateway (Lists & Rules).
    - `next_dns`: Интеграция с NextDNS (Rewrites & Deny).

## 📝 Особенности разработки
- **Anti-Regression**: При обновлении Cloudflare скрипт удаляет старые списки/правила, созданные им ранее (идентифицирует по префиксам и `Session ID` в описании).
- **NextDNS API**: Имеет жесткие лимиты (60 запросов в минуту), поэтому в коде реализована обработка `429 Too Many Requests`.
- **Java 25 Preview**: Весь проект написан с использованием экспериментальных возможностей Java 25, что требует флага `--enable-preview` при компиляции и запуске.

## 🧪 Тестирование
Специфические Unit-тесты в структуре отсутствуют, основная проверка — запуск через GitHub Actions. Для локальной отладки рекомендуется использовать `REDIRECT` и `BLOCK` с небольшими файлами.
