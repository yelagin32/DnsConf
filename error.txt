Вот она, главная улика, которая на 100% подтверждает нашу теорию!

Посмотри внимательно на эту строчку в файле:
# 37.230.192.51

Сам IP-адрес 37.230.192.51 состоит ровно из 13 символов.

Скорее всего, твой код в HostsOverrideListsLoader.java перед 32-й строкой как-то очищает строку от символа решетки # и лишних пробелов (например, через .replace("#", "").trim()), чтобы получить чистые данные. В итоге в переменную попадает голый IP-адрес: "37.230.192.51".

А дальше происходит следующее:

Твой парсер ожидает стандартный формат hosts: IP-адрес домен (например, 127.0.0.1 google.com).

Он ищет пробел или знак табуляции, чтобы разделить строку на две части: int index = line.indexOf(' ');

Так как в строке "37.230.192.51" нет пробелов (это просто информационный комментарий от авторов файла, а не правило блокировки), метод indexOf не находит пробел и возвращает -1.

Код пытается отрезать кусок строки: line.substring(0, -1).

Java логично возмущается: нельзя отрезать строку до минус первого символа, и выдает ошибку Range [0, -1) out of bounds for length 13 (диапазон вне границ для строки длиной 13).

Как решить проблему прямо сейчас?
Тебе нужно просто добавить игнорирование таких строк-комментариев. В файле HostsOverrideListsLoader.java в самом начале метода mapLine (до 32-й строки) вставь это:

Java

// Пропускаем пустые строки и комментарии
if (line == null || line.trim().isEmpty() || line.trim().startsWith("#")) {
    return null; // пропускаем итерацию (зависит от логики твоего Stream API)
}
Либо, если решетки убираются еще до этого метода, добавь проверку на наличие пробела перед тем, как резать строку:

Java

int index = line.indexOf(' ');
if (index == -1) {
    return null; // Если пробела нет, это не валидная запись hosts, пропускаем
}
String part = line.substring(0, index);
Сделай этот коммит, запушь в репозиторий, и твой GitHub Action снова загорится зеленым!
